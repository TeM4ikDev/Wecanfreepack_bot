# Настройка окружений

## Простая система конфигурации для разных окружений

### Файлы окружения

- `env.development` - для разработки
- `env.production` - для продакшена  
- `env.testing` - для тестирования
- `env.example` - шаблон конфигурации

### Как использовать

#### 1. Настройка окружения
```bash
# Для разработки
npm run env:dev

# Для продакшена
npm run env:prod

# Для тестирования
npm run env:test
```

#### 2. Запуск приложения
```bash
# Разработка
npm run start:dev

# Продакшен
npm run start:prod

# Тестирование
npm run start:test
```

#### 3. Полный цикл для продакшена
```bash
npm run env:prod
npm run start:prod:build
```

### Структура конфигурации

Каждый файл окружения содержит:
- `NODE_ENV` - текущее окружение
- `DATABASE_URL` - строка подключения к БД
- `TELEGRAM_BOT_TOKEN` - токен Telegram бота
- `JWT_SECRET` - секретный ключ для JWT
- `PORT` - порт сервера
- `LOG_LEVEL` - уровень логирования

### Важные моменты

1. **Никогда не коммитьте файлы .env** - они содержат секретные данные
2. **Используйте env.example** как шаблон для создания новых окружений
3. **Для продакшена** обязательно смените все секретные ключи
4. **Для тестирования** используйте отдельную БД

### Автоматическое определение окружения

Приложение автоматически определяет окружение по переменной `NODE_ENV`:
- Если `NODE_ENV=development` → загружается `.env.development`
- Если `NODE_ENV=production` → загружается `.env.production`  
- Если `NODE_ENV=testing` → загружается `.env.testing`
- По умолчанию → загружается `.env.development`
